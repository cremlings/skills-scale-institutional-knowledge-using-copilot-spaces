# OctoAcme Project Management Processes

Welcome to the OctoAcme Project Management documentation hub. This README provides a comprehensive overview of our project management processes and links to detailed guidance for each phase of the project lifecycle.

## Overview of OctoAcme Project Management

OctoAcme follows a structured, customer-first approach to project delivery. Our methodology emphasizes:

- **Customer-first mindset**: Prioritizing customer value and usability in all decisions
- **Iterative delivery**: Delivering small, testable increments to gather feedback early
- **Clear ownership**: Every project has named Project Manager and Product Lead roles
- **Data-informed decisions**: Measuring impact and iterating based on evidence
- **Psychological safety**: Encouraging feedback, learning, and continuous improvement

### Core Principles
Our processes are built on five foundational principles:
1. **Clarity** - Well-defined roles, responsibilities, and decision gates
2. **Transparency** - Regular communication and shared visibility into project status
3. **Efficiency** - Minimizing overhead while maximizing delivery impact
4. **Accountability** - Clear ownership and escalation paths
5. **Continuous Improvement** - Learning from each project to refine our approach

## Project Lifecycle

OctoAcme projects follow a standardized lifecycle with five key phases:

```
Initiation → Planning → Execution → Release → Close & Retrospective
```

## Key Artifacts

Throughout your project, you'll create and maintain these core artifacts:

- **Project Charter / One-pager**: Business case and success criteria
- **Backlog**: Prioritized list of work with acceptance criteria
- **Sprint/Iteration Backlog**: Items committed for the current sprint
- **Definition of Done**: Shared understanding of what "complete" means
- **Risk Register**: Ongoing tracking of identified risks and mitigations
- **Release Notes**: Summary of changes and known issues
- **Retrospective Notes**: Learnings and action items for improvement

## Communication Cadence

- **Weekly PM + PdM Sync**: Alignment on priorities, risks, and decisions
- **Twice-Weekly Standups**: Delivery team progress and blocker triage
- **Monthly Stakeholder Updates**: High-level status and key achievements
- **Ad-hoc Escalations**: For critical blockers or decisions needed
